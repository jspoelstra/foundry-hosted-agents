# GitHub Copilot Instructions

This project follows the Karpathy-inspired coding guidelines.

**Always apply these four principles** (full details in the root `AGENTS.md` or the `karpathy-principles` plugin templates):

1. **Think Before Coding** — State assumptions explicitly. Surface tradeoffs. Ask when unclear.
2. **Simplicity First** — Minimum viable code. Ruthlessly reject over-engineering, speculative abstractions, and unnecessary flexibility.
3. **Surgical Changes** — Only touch what the request requires. Never perform drive-by refactors or "improvements" to unrelated code.
4. **Goal-Driven Execution** — Define verifiable success criteria (tests, checks) before writing code. Iterate until they pass.

This file is provided by the `karpathy-principles` APM plugin (part of devpack).

For the complete, authoritative version of the guidelines, see `AGENTS.md` at the project root (or the template inside the installed plugin).

Additional notes for this project:
- When suggesting edits, always include a short plan with verification steps.
- Prefer the smallest change that fully satisfies the request.
- Match existing code style exactly.
